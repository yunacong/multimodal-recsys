### 关键工程优化 (1222x 加速)

**第一版实现**(慢):
```python
for each positive sample:
    np.random.choice(...)  # 410 万次调用!
# 速度: 596 正样本/秒
# 预计总耗时: 1.8 小时
```

**第二版实现**(快):
```python
# 一次性抽样海量候选
candidates = np.random.choice(items, size=N*5*3, replace=True)
# 然后 Python loop 只做"过滤+组装"
# 速度: 728,551 正样本/秒
# 实际耗时: 5.6 秒
```

**优化原理**: numpy 哲学 —— 能向量化就不要 Python loop。
np.random.choice 单次调用有 ~250μs 初始化开销,
410 万次调用 = 1000 秒纯开销。
改成"一次抽 6000 万 + 向量化过滤"消除这个开销。

**加速比: 1222x** ⭐

## 任务 2: 八股 #1 - 逻辑回归

### 三大组件

1. **线性部分**: score = w·x + b, 范围 (-∞, +∞)
2. **Sigmoid**: 把 score 挤压到 (0, 1), 解释成概率
3. **BCE 损失**: 衡量"预测概率"和"真实标签"的差距

### 关键洞察(今天的学习历程)

我答错了 3 道选择题:
- "线性回归不能做分类的原因" 错选 "不能多类分类"
- "sigmoid 输出范围" 错选 (-∞, +∞)
- "0.5 阈值固定?" 错选 "固定"

通过反向教学(从损失函数倒推 sigmoid 必要性)+ 银行贷款类比
+ BCE 数值实验,最终建立完整理解链条:

```
特征 x → score = w·x+b → sigmoid → 概率 p
                                    ↓
                              和真实 y 算 BCE 损失
                                    ↓
                              反向传播更新 w, b
```

### BCE 的"非对称惩罚"特性

- 真实 y=1, 预测 p=0.99 → 损失 0.010(几乎无惩罚)
- 真实 y=1, 预测 p=0.001 → 损失 6.91(巨大惩罚)
- 差异比 MSE 大得多 → 模型学得快

**现象**: 粘贴含 `# push` 的命令时报 `command not found: #`

**教训**: shell 之间的细节差异要警惕(bash vs zsh 默认行为不同)
