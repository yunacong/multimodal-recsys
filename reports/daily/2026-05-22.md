## 踩坑记录

### 坑 #1: HuggingFace 下载得到 15 字节"假文件"

- **现象**: curl 显示 100% 完成,但文件只有 15 字节,平均速度仅 79 B/s
- **原因**: URL 后缀加了 .gz,但 HuggingFace 上实际文件是未压缩的 .jsonl
- **修复**: 去掉 .gz 后缀,使用正确 URL 重新下载,得到完整 311 MB 文件
- **教训**: 下完文件第一时间用 `ls -lh` 和 `head -1` 验证大小和内容,
  不要假设"100%"就是成功

### 坑 #2: Apple Silicon Mac 上 LightGBM 缺 OpenMP

- **现象**: `import lightgbm` 报错 `Library not loaded: @rpath/libomp.dylib`
- **原因**: macOS Clang 编译器不自带 OpenMP 运行时,
  LightGBM C++ 内核依赖 OpenMP 做并行计算
- **修复**: `brew install libomp`
- **教训**: Apple Silicon 环境配置要预留时间处理系统库依赖,
  这是非 Linux 平台跑 ML 框架常见问题

### 坑 #3: Heredoc 嵌套 EOF 卡死终端

- **现象**: cat > README.md << 'EOF' ... EOF 写入卡在 heredoc> 提示符
- **原因**: README 内容里嵌套了代码块,其中又包含 EOF 字样,导致 shell 误判
- **修复**: 改用 EOF_README_END 这种不易冲突的分隔符,同时简化 README 内容
- **教训**: heredoc 分隔符要选不可能在内容里出现的字符串

### 坑 #4: GitHub 网页创建仓库后本地 push 冲突风险

- **现象**: 网页创建时勾选了 LICENSE,远程有了 1 个 commit;本地若直接 init+push 会被拒
- **修复**: 先 git clone 远程仓库到临时目录,把 .git 和 LICENSE 搬到本地项目目录,
  再添加本地新文件,避免历史分叉
- **教训**: GitHub 网页创建仓库时,要么完全空仓库,要么准备好处理远程已有 commit 的情况

## 关键决策

- **商品标识**: 使用 parent_asin 而非 asin,避免同款不同 SKU(颜色/规格)被算作多个商品

## 今日所学

- **虚拟环境的本质**: 隔离不同项目的 Python 解释器和依赖,避免版本冲突
- **.gitignore 必须先于 git add 创建**: 否则文件已进入暂存区,需要 git rm --cached 才能移除
- **Git 三个区**: 工作区 → 暂存区(add) → 本地仓库(commit) → 远程仓库(push)
- **macOS 与 Linux 的差异**: Clang 不自带 OpenMP,需 brew 单独安装 libomp
- **数据敏感度**: 永远验证下载文件的大小和内容,不要相信"100%"
